# RAG测试用例生成示例（数据优先版）

本文档展示了如何使用**数据优先**的RAG测试用例生成智能体。

**核心原则**：优先检索原始结构化数据，而非LLM生成的回答。

---

## 使用方式

### 1. 启动RAG MCP服务器

```bash
python src/examples/rag_mcp_server.py --transport sse --port 8008
```

### 2. 运行测试用例生成智能体

```python
from src.examples.rag_agent import agent

# 智能体会自动按优先级使用RAG工具
result = agent.invoke({"messages": [
    ("human", "请帮我生成现付订单相关的测试用例")
]})
```

### 3. 智能体工作流程（数据驱动）

```
Step 1: 使用 rag_query_data 检索原始实体和关系
        → 获取 entities, relations, chunks

Step 2: 使用 rag_graph_get 获取关键实体的关联图
        → 了解实体间的完整关系网络

Step 3: 整理数据结构
        - 实体清单（名称、属性、约束规则）
        - 状态机（状态定义、转换条件）
        - 业务规则（计算公式、校验逻辑）

Step 4: 基于原始数据推导测试场景
        - 从实体属性推导测试数据
        - 从关系定义推导流程场景
        - 从规则描述推导验证点

Step 5: 生成可追溯的测试用例
        - 每条用例标注数据来源
        - 预期结果基于原始规则文本
```

## RAG工具使用策略对比

### ❌ 不推荐：仅使用 rag_query

```python
# 不推荐的方式 - 依赖LLM生成
rag_query(query="现付订单的退款规则是什么？")

# 问题：
# 1. 回答可能不准确（幻觉）
# 2. 无法追溯来源
# 3. 可能遗漏关键细节
```

### ✅ 推荐：优先使用 rag_query_data

```python
# 推荐的方式 - 检索原始数据
rag_query_data(
    query="现付订单 退款规则 预收抵扣",
    mode="hybrid",
    top_k=100,
    chunk_top_k=10,
    include_chunk_content=True
)

# 优势：
# 1. 直接获取知识库原始内容
# 2. 可追溯到具体文档和章节
# 3. 包含完整的规则描述
```

## 最佳实践总结

### 1. 始终优先使用原始数据

```python
# ✅ 正确顺序
1. rag_query_data()     # 获取原始实体和规则
2. rag_graph_get()      # 获取关联关系
3. rag_query()          # 仅在概念理解时使用

# ❌ 错误顺序
1. rag_query()          # 不要先依赖LLM生成
2. rag_query_data()     # 只作为验证
```

### 2. 合理使用检索参数

```python
# 获取详细规则
rag_query_data(
    query="退款规则",
    mode="hybrid",          # 混合检索
    top_k=100,              # 更多实体
    chunk_top_k=10,         # 更多文本块
    include_chunk_content=True  # 必须包含内容
)

# 精确查找实体
rag_query_data(
    query="现付订单",
    mode="local",           # 局部精确
    top_k=20
)
```

### 3. 验证数据来源

每条测试用例应包含数据溯源：
```markdown
**数据溯源**:
- 实体定义: rag_query_data → entities[0]
- 规则描述: rag_query_data → chunks[5]
- 关联关系: rag_graph_get → "现付订单"子图
```

### 4. 数据不足时明确告知

不要基于假设生成大量用例，应：
1. 明确列出缺失的数据
2. 建议补充的文档
3. 基于合理假设生成（标注为待确认）

## 与LLM生成式查询的对比

| 场景 | 推荐工具 | 原因 |
|------|---------|------|
| 获取字段定义和规则 | `rag_query_data` | 需要精确、可追溯的数据 |
| 了解实体关系 | `rag_graph_get` | 需要完整的关联网络 |
| 理解业务术语定义 | `rag_query` | 概念解释，容错性高 |
| 获取流程概述 | `rag_query` | 高层次理解，细节可后续验证 |
| 设计测试数据 | `rag_query_data` | 必须基于精确的字段规则 |
| 设计验证点 | `rag_query_data` | 必须基于原始规则描述 |
